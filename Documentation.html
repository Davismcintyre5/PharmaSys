Introduction

PharmaSys is a comprehensive, full‑stack pharmacy management system designed to streamline the daily operations of a modern pharmacy. Built with the MERN stack (MongoDB, Express.js, React, Node.js), it offers a robust, secure, and user‑friendly interface for managing inventory, patients, prescriptions, financial accounts, invoices, withdrawals, and more. The system is deployed and accessible online, demonstrating its production readiness.

This document provides a detailed overview of the system's architecture, features, technology stack, API endpoints, deployment procedures, and future roadmap. It serves as both a technical reference for developers and a project summary for stakeholders.

1. Features

  • Dashboard – Real‑time overview: current balance, today's revenue/expenses, low stock alerts, patient counts, and 7‑day income chart.
  • Inventory Management – Full CRUD for medicines, batch tracking, expiry alerts, and low stock notifications.
  • Patient Management – Manage patient profiles, contact details, and prescription history.
  • Prescription Handling – Create, view, and fulfill prescriptions with stock auto‑deduction; creates pending transactions.
  • Supplier Management – Keep detailed records of medicine suppliers.
  • Accounts & Approval Workflow – Transactions require approval before affecting balance; admin/manager can approve/reject pending transactions.
  • Invoices – Create invoices for patients (sale) or suppliers (purchase); mark paid creates pending transaction.
  • Withdrawals – Request withdrawals; approval creates pending expense transaction.
  • Reporting Module – Sales and inventory reports with custom date ranges; print with pharmacy watermark.
  • Print Functionality – All lists and individual records have print buttons with footer (pharmacy name, generation details).
  • User & Settings Management – Admin panel for user CRUD, role assignment, and pharmacy configuration (name, address, currency, tax rate).
  • Role‑Based Access Control – Secure routes based on roles: admin, manager, pharmacist, cashier.

3. System Architecture

The system follows a client‑server architecture with a clear separation between frontend and backend:

  • Client (React App): Served as static files. All dynamic data is fetched via API calls.
  • Server (Node.js/Express): Exposes RESTful API endpoints, handles authentication, business logic, and database interactions.
  • Database (MongoDB): Stores all application data in collections: users, patients, medicines, prescriptions, suppliers, transactions, invoices, withdrawals, settings.

Communication is stateless; each request includes a JWT token for authentication. Transactions require approval workflow; only completed transactions affect financial summaries.

6. Security Considerations

  • Authentication: JWT tokens with expiration; passwords hashed with bcrypt.
  • Authorization: Role‑based middleware protects routes.
  • CORS: Configured to allow only trusted origins.
  • Input Validation: Request data validated via Mongoose schemas.
  • Environment Variables: Sensitive keys stored in .env, never committed.
  • Helmet.js: Sets secure HTTP headers in production.
  • Rate Limiting: 100 requests per 15 minutes per IP.
  • HTTPS: All traffic encrypted in production.

7. Deployment Guide

Backend

  1. Set NODE_ENV=production and configure environment variables.
  2. Use process manager: pm2 start server.js --name pharmasys-backend
  3. Set up reverse proxy with SSL.

Frontend

  1. Set VITE_API_URL=https://pharmasysserver.pxxl.click/api
  2. Build: npm run build
  3. Upload dist/ folder to static hosting.

Both services are currently live at the URLs provided.
