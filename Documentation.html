Introduction

PharmaSys is a comprehensive, full‑stack pharmacy management system designed to streamline the daily operations of a modern pharmacy. Built with the MERN stack (MongoDB, Express.js, React, Node.js), it offers a robust, secure, and user‑friendly interface for managing inventory, patients, prescriptions, financial accounts, and more. The system is deployed and accessible online, demonstrating its production readiness.

This document provides a detailed overview of the system's architecture, features, technology stack, API endpoints, deployment procedures, and future roadmap. It serves as both a technical reference for developers and a project summary for stakeholders.

1. Features

  • Dashboard – Real‑time overview of key metrics: current balance, today's revenue/expenses, low stock alerts, patient counts, and a 7‑day income chart.
  • Inventory Management – Full CRUD for medicines, batch tracking, expiry alerts, and low stock notifications.
  • Patient Management – Manage patient profiles, contact details, and prescription history.
  • Prescription Handling – Upload, verify, and manage prescriptions with status tracking (pending, approved, dispensed, rejected).
  • Supplier Management – Keep detailed records of medicine suppliers, including contact persons and addresses.
  • Financial Accounts – Record income, expenses, and withdrawals; view current balance; print individual transaction receipts (with pharmacy name watermark).
  • Reporting Module – Generate and print sales, inventory, and financial reports with custom date ranges; all printouts include a subtle pharmacy watermark.
  • User & Settings Management – Admin panel for managing system users (CRUD) and configuring pharmacy details, currency, and tax rate.
  • Role‑Based Access Control – Secure routes based on user roles: admin, pharmacist, cashier, manager.
  • Notifications – In‑app alerts for low stock, expiring medicines, and prescription updates.

2. Technology Stack

Frontend (React)

  • React 18 – Functional components, hooks, and context API.
  • React Router v6 – Client‑side routing with hash‑based strategy for easy static hosting.
  • Redux Toolkit – State management for user authentication and global UI state.
  • Formik & Yup – Form handling and validation.
  • Chart.js & react-chartjs-2 – Interactive dashboard charts.
  • Axios – HTTP client for API communication.
  • CSS3 – Fully custom responsive design (no UI framework).

Backend (Node.js + Express)

  • Node.js – JavaScript runtime.
  • Express.js – Web framework for building RESTful APIs.
  • MongoDB + Mongoose ODM – Data modeling and persistence.
  • JWT (JSON Web Tokens) – Authentication and authorization.
  • Bcrypt.js – Password hashing.
  • Multer – File uploads (prescription images).
  • Node‑cron – Scheduled tasks (e.g., expiry alerts).
  • Nodemailer / Twilio – Email/SMS notifications (optional, configured via environment).
  • Winston – Logging.
  • CORS – Configured for multiple origins (local and production).

3. System Architecture

The system follows a classic client‑server architecture with a clear separation between frontend and backend:

  • Client (React App): Served as static files from a web server (e.g., Nginx). All dynamic data is fetched via API calls to the backend.
  • Server (Node.js/Express): Exposes RESTful API endpoints, handles authentication, business logic, and database interactions.
  • Database (MongoDB): Stores all application data in collections: users, patients, medicines, suppliers, prescriptions, transactions, settings, notifications.

Communication between client and server is stateless; each request includes a JWT token for authentication. The frontend uses hash‑based routing, allowing it to be hosted on any static server without special configuration.

6. Security Considerations

  • Authentication: JWT tokens with expiration; passwords hashed with bcrypt.
  • Authorization: Role‑based middleware protects routes (e.g., admin‑only endpoints).
  • CORS: Configured to allow only trusted origins (localhost, live domain).
  • Input Validation: Request data validated using express‑validator and Mongoose schemas.
  • Environment Variables: Sensitive keys (JWT secret, database URI) stored in .env, never committed.
  • Helmet.js: (Optional) Used to set secure HTTP headers.
  • Data Encryption: All traffic over HTTPS in production.

7. Deployment Guide

Backend

  1. Set NODE_ENV=production and configure environment variables on the server.
  2. Use a process manager like PM2: pm2 start server.js --name pharmasys-backend
  3. Set up a reverse proxy (e.g., Nginx) to forward requests to port 5000 and enable SSL.

Frontend

  1. Build the React app: npm run build (ensure REACT_APP_API_URL points to the live backend).
  2. Upload the contents of the build/ folder to your web server's document root.
  3. Configure the server to serve index.html for all routes (for client‑side routing). Example Nginx:
location / {
    try_files $uri /index.html;
}

Both services are currently live at the URLs provided.
